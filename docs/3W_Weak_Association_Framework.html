3W Weak Association Framework

Where / Why / When under weak evidence：在弱拓扑、弱因果、弱时序条件下，恢复可解释的事件演化链

◎ 核心定义

3W 的本质，是以多维时序点过程为技术底座，用 Where、Why、When 三类弱证据，把离散事件恢复成可解释的事件演化链。

▣ 英文统一口径
  • Where = Weak Topological Association
  • Why = Weak Causal Association
  • When = Weak Temporal Association

一、3W 命名对齐关系

把原来的 Weak Topology / Causality / Temporal 与 Where / Why / When 对齐
W

Where

Weak Topological Association

对应原来的 Weak Topology。更地道的表达是“弱拓扑关联”，强调事件是否处于同一影响域，以及弱拓扑上的事件演化关系。

W

Why

Weak Causal Association

对应原来的 Weak Causality。注意不是 Casality，正确拼写是 Causality。这里建议用“Causal Association”，更贴合事件链解释。

W

When

Weak Temporal Association

对应原来的 Weak Temporal。单独说 Weak Temporal 不够完整，建议补成“弱时序关联”，强调事件之间的时间触发模式。

推荐统一表述：Our 3W framework studies event association from three weak-evidence dimensions: Where for weak topological association, Why for weak causal association, and When for weak temporal association.

二、为什么需要 3W？

从“强规则关联”走向“弱证据融合”

传统方法的强假设

  • 精确拓扑：设备、链路、端口、业务承载关系完整可信。
  • 确定规则：专家规则可以稳定描述事件之间的关系。
  • 固定时序：事件按照稳定顺序和固定时间窗口出现。

真实网络中的弱条件

  • 拓扑弱：链路方向缺失、业务承载不清、拓扑关系不完整。
  • 因果弱：中间事件缺失、噪声事件混入、根因不可见。
  • 时序弱：告警延迟、乱序、批量爆发、采集周期不一致。
复杂网络运维的核心挑战，不是缺少告警，而是缺少把离散告警组织成事件演化链的能力。

技术底座：多维时序点过程把 3W 统一到同一个强度函数中

参考 MHP 建模页的表述，一段事件流可以写成 \(\mathcal{D}=\{e_i=(t_i,u_i,m_i)\}\)：\(t_i\) 是时间，\(u_i\) 是事件类型，\(m_i\) 是设备、位置、业务、状态等上下文。3W 的三个 W 不是三套割裂规则，而是共同影响目标类型 \(u\) 在时刻 \(t\) 的发生强度。

\[ \lambda_u(t) = \mu_u + \sum_{j:t_j\lt t} \alpha_{u,u_j}(t) \cdot g_{u,u_j}(t - t_j) \]

Where 进入 α

拓扑、位置、实体属性、业务承载被编码为 \(\phi_{\mathrm{static}}(u,v)\)，并通过 \(r_{\mathrm{topo}}(v\rightarrow u)\) 与结构先验影响触发强度。

Why 解释父子关系

每个事件要么由背景强度 \(\mu\) 解释，要么由历史事件触发；父事件变量 \(z_i\) 把“谁影响了谁”显式化。

When 进入 g

时间衰减核 \(g(\Delta t)\) 让历史影响随时间平滑减弱，并用 time slack 对小范围乱序进行折扣关联。

Context 动态修正

源设备未恢复告警、目标设备脆弱状态、链路方向和专家知识共同组成动态上下文 \(\mathbf{m}_{u,v}(t)\)。

五、3W 融合推理公式

统一关联置信度
◇ Association Score
\[ S(e_i, e_j) = \alpha S_{\mathrm{where}} + \beta S_{\mathrm{why}} + \gamma S_{\mathrm{when}} + \delta S_{\mathrm{context}} \]

单一弱证据不可靠，多维弱证据融合形成稳定解释。

公式中的四个分量

  • \(S_{\mathrm{where}}\)弱拓扑关联分：来自 \(r_{\mathrm{topo}}(v\rightarrow u)\)、同站点/同实体/业务承载特征，以及拓扑 pseudo-count 对稀疏样本的温和支持。
  • \(S_{\mathrm{why}}\)弱因果解释分：来自父事件分配 \(z_i\)、缺失事件插补收益、解释链完整性和后验置信度。
  • \(S_{\mathrm{when}}\)弱时序触发分：来自 \(\alpha\cdot g(\Delta t)\)、延迟分布、时间方向和乱序松弛折扣。
  • \(S_{\mathrm{context}}\)场景上下文分：设备角色、链路方向、运行状态、专家知识是否支持当前关联。

Context 是什么？

Context 就是场景背景和工程约束。

Where / Why / When 判断“像不像相关”，Context 判断“放在这个场景里合不合理”。

  • 设备角色：核心、汇聚、接入、传输等。
  • 链路方向：上行、下行、主备链路、保护路径。
  • 业务承载：同一客户、同一区域、同一专线或同一业务域。
  • 知识约束：专家规则、历史模式、故障基因库、典型场景库。

从强度函数到关联分数

在实现层面，关联分数可以看作对点过程触发项的工程化归一：先学习 \(\alpha_{u,v}(t)\)，再用时间核计算实例级触发分数，最后叠加弱拓扑、弱因果和上下文校正。

\[ s(j\rightarrow i) = \alpha_{u_i,u_j}(t_i) \cdot g(t_i - t_j) \]
  • 训练目标：最大化观测事件的发生强度，同时通过积分惩罚避免模型在无事件区间过度预测。
  • 软责任：\(P(z_i=j)\) 可由候选父事件触发项占总强度的比例得到，用于统计类型对触发次数。
  • 稀疏校正：\(\widetilde{N}_{u,v}=N_{u,v}+\rho\cdot r_{\mathrm{topo}}(v\rightarrow u)\) 让弱拓扑只提供软偏置，不变成硬规则。

六、为什么不同问题选择不同方法？

方法要匹配问题形态

Where 用 Diffusion

弱拓扑本质是大规模影响域和关联强度学习问题。拓扑不直接决定因果边，而是转为可学习的结构特征、候选空间和先验强度。

Why 用可逆 MCMC

弱因果本质是少量关键隐变量的结构补全问题。可逆 MCMC 适合在固定滞后窗口内增加、删除、替换潜在事件，并同步重采样父子关系。

When 用点过程

弱时序本质是离散事件流中的时间触发关系学习。点过程关注背景强度、历史触发强度、连续时间衰减和小范围乱序鲁棒性。

三类方法在在线推理中的协作

候选父事件选择

新事件到达后，系统在 Where 限定的候选集合中比较 \(\mu\) 与各个 \(\alpha \cdot g\)，决定它是开新链还是并入已有链。

缺失事件插补

当目标事件无法被已观测父事件合理解释时，Why 在窗口内提出潜在中间事件，并用解释收益、额外预测惩罚和缺失先验共同决定是否保留。

时间松弛提交

When 维护 pending 区域和 watermark，只在超过松弛范围后提交父子关系，避免因采集延迟或乱序到达过早定案。

置信度汇总

窗口内多轮采样会形成父事件投票；提交时输出最可信父节点、是否插补、链路置信度和工程上下文解释。

核心判断：弱拓扑不是“修几条边”，而是学习影响域如何扩展；弱因果不是“调几个参数”，而是补全事件链中的缺失环节；弱时序不是“看谁先出现”，而是学习事件之间的时间触发模式。

八、对外汇报口径

可直接用于客户材料

We propose a 3W Weak Association Framework to understand event association in complex network operations from three weak-evidence dimensions, using a multivariate temporal point process as the modeling backbone.

  • Where — Weak Topological Association: identifies whether events belong to the same impact domain by using topology as learnable features and soft priors.
  • Why — Weak Causal Association: assigns parent events, completes missing events in the event evolution chain, and outputs probabilistic causal explanations.
  • When — Weak Temporal Association: learns continuous-time triggering patterns, delay decay, and time-slack associations between events.
  • Context: uses device roles, service context, dynamic states, expert knowledge, and the fault gene library to calibrate association decisions.

Together, these dimensions organize isolated alarms into explainable event evolution chains, supporting fault grouping, impact domain identification, root cause reasoning, and scenario classification.

中文一句话：3W 的本质，是用 Where、Why、When 三类弱证据，结合场景上下文，把离散告警恢复成可解释的事件演化链。
