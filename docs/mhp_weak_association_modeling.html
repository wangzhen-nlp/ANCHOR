多维时序点过程 · 弱关联故障建模

以多维时序点过程为骨架，提出弱拓扑、弱因果、弱时序三类「弱关联」建模。拓扑、结构与时序都不作硬规则，而是作为可学习的归纳偏置进入触发强度——真正的关系，由事件数据与学习到的参数共同决定。

Section 01 · Multivariate Temporal Point Process

第一节：多维时序点过程

我们将一段连续发生的事件序列建模为一个多维连续时间点过程。模型的核心问题是：当一个新事件出现时，它究竟是独立产生的，还是受到了历史事件的触发。

1.1 事件流表示

给定一段观测到的事件流：

\[\mathcal{D}=\{e_i=(t_i,u_i,m_i)\}_{i=1}^{N}\]

其中，\(t_i\) 表示事件发生时间，\(u_i\) 表示事件所属类型，\(m_i\) 表示事件的附加属性或上下文信息。 从建模角度看，\(u_i\) 决定了事件属于哪一个维度，\(t_i\) 决定了事件之间可能存在的时序影响。

1.2 发生强度：背景项 + 历史触发项

对任意事件类型 \(u\)，模型用一个时间相关的强度函数 \(\lambda_u(t)\) 描述该类型事件在时刻 \(t\) 发生的倾向：

\[\lambda_u(t)=\mu_u+\sum_{j:t_j\lt t}\alpha_{u,u_j}\,g_{u,u_j}(t-t_j)\]

第一项 \(\mu_u\) 表示该类型事件的背景发生倾向，也就是不依赖历史事件时自身出现的基础强度。 第二项表示历史事件对当前类型的触发作用：历史事件越相关、时间越接近，对当前事件的影响通常越强。

1.3 学习目标：解释已发生事件，同时避免过度预测

在强度函数定义好之后，模型训练的目标是最大化观测事件流的似然：

\[\log p(\mathcal{D})=\sum_{i=1}^{N}\log\lambda_{u_i}(t_i)-\sum_{u}\int_{0}^{T}\lambda_u(t)\,dt\]

第一项鼓励模型在真实事件发生的时刻给出较高强度；第二项是积分惩罚，避免模型在大量没有事件发生的时间段也给出过高强度。 因此，模型不仅要解释已经发生的事件，还要控制整体预测不要过度扩散。

1.4 学习方法：EM 算法与隐含父变量

直接优化上述似然并不方便，因为每个事件的来源并没有被观测到。为此，学习过程中引入隐含父变量 \(z_i\)，它不是最终建模目标， 而是 EM 算法中的中间推断量，用来辅助估计参数。

\[z_i=0 \quad \text{表示事件 } e_i \text{ 由背景项解释}\] \[z_i=j \quad \text{表示事件 } e_i \text{ 由历史事件 } e_j \text{ 触发}\]

在 E-step 中，模型根据当前参数计算不同来源解释当前事件的相对责任：

\[P(z_i=0\mid\mathcal{D})=\frac{\mu_{u_i}}{\lambda_{u_i}(t_i)}\] \[P(z_i=j\mid\mathcal{D})=\frac{\alpha_{u_i,u_j}g_{u_i,u_j}(t_i-t_j)}{\lambda_{u_i}(t_i)}\]

在 M-step 中，模型固定这些软责任，重新最大化上一节的学习目标：背景项会吸收被分配给 \(z_i=0\) 的期望计数，触发项会吸收被分配给 \(z_i=j\) 的期望触发计数；同时，积分惩罚项会约束背景强度和触发强度不要无限增大。宏观上，M-step 就是在用 E-step 得到的软分配结果，更新 \(\mu\)、\(\alpha\) 以及时间衰减相关参数。

1.5 宏观流程

1
输入事件流
将事件按时间排列，每个事件包含时间、类型和上下文信息。
2
定义强度
用背景项和历史触发项描述不同类型事件在连续时间上的发生倾向。
3
优化似然
通过似然目标同时解释已发生事件，并惩罚过度预测。
4
EM 学习
将隐含父变量作为中间产物，在软分配和参数更新之间迭代。

1.6 小结

本节只介绍多维时序点过程的宏观思想：先用背景强度、触发强度和时间衰减函数定义事件发生强度，再通过似然目标学习参数；隐含父变量只是 EM 学习过程中的中间推断量，用于辅助判断每个事件更可能由背景项还是历史事件解释。

1.7 参考文献：相关论文的核心思想

本节的多维时序点过程建模思路，主要借鉴了四类相关工作：ISAHP 强调在实例级别解释“哪个历史事件影响了当前事件”，BRUNCH 强调从点过程里恢复分支结构，DHP 强调用 Hawkes 强度驱动连续时间流式聚类，Latent Network Hawkes 强调从事件流中发现隐含网络。它们共同指向一个核心思想：点过程不仅能建模事件发生强度，还能进一步解释事件之间的弱关联结构。

参考一：ISAHP——实例级 Granger 因果解释

出处：Dongxia Wu、Yi-An Ma、Rose Yu（University of California, San Diego）；Tsuyoshi Idé、Aurélie Lozano、Georgios Kollias、Jiří Navrátil、Naoki Abe（IBM T. J. Watson Research Center）. Learning Granger Causality from Instance-wise Self-attentive Hawkes Processes, AISTATS 2024。

该工作关注异步、多类型事件序列中的实例级 Granger 因果发现。传统 Hawkes 过程通常得到类型级因果矩阵，而 ISAHP 希望直接解释具体事件实例之间的影响关系。

\[ \lambda(x\mid H_t) = \mu(x\mid H_t) + \sum_{j:t_j\lt t}\alpha(x,x_j)\,\phi(t-t_j\mid x,x_j) \]

ISAHP 的关键是保留 Hawkes 强度函数的逐项加和结构：每个历史事件仍然单独贡献一项影响，因此实例级影响 \(\alpha(x,x_j)\) 可以直接作为事件 \(j\) 对当前事件的因果强度解释。

\[ A(x,x_j)= \frac{\exp(x^{\top}Kx_j)\,\mathbf{1}(t>t_j)} {\sum_{l:t_l\lt t}\exp(x^{\top}Kx_l)} \]
\[ \alpha(x,x_j)=\sigma_{+}\!\bigl(A(x,x_j)\,(w_k^{\alpha})^{\top}v_j\bigr) \]

算法流程：先把事件类型和时间差编码为事件 embedding；再通过 self-attention 计算当前事件与历史事件之间的相关权重；随后用注意力加权后的历史事件表示参数化背景强度、实例级影响强度和时间衰减率；最后通过最大似然训练，使 \(\alpha(x,x_j)\) 成为可直接读取的实例级影响分数。

对当前方法的启发：当前方法虽然没有采用神经自注意力，但同样保留了“当前事件由多个历史事件逐项贡献强度”的可解释结构；这使得在线推理中的父事件选择、插补链路和弱因果解释都有明确的实例级含义。

参考二：BRUNCH——分支结构推断

出处：Hui Li、Sourav S. Bhowmick（Nanyang Technological University, Singapore）；Hui Li（Xidian University, China）. BRUNCH: Branching Structure Inference of Hybrid Multivariate Hawkes Processes with Application to Social Media, PAKDD 2020。

该工作关注 Hybrid Multivariate Hawkes Processes 中的 branching structure 推断。论文认为，多维点过程中的事件可以分为 immigrant 与 offspring：immigrant 表示独立产生的根事件，offspring 表示由历史事件触发的后继事件；把这些父子触发关系连接起来，就得到一条或多条 cascade。

  • \(Z_{ik}=t_{ik}\)：表示事件 \(t_{ik}\) 是 immigrant（独立产生的根事件）。
  • \(Z_{ik}=t_{jl}\)：表示事件 \(t_{jl}\) 触发了事件 \(t_{ik}\)（父子触发关系）。

BRUNCH 的关键做法是使用 intensity-driven Chinese Restaurant Process（intCRP）对分支结构建模：先在单个维度内部构造 event links，再在不同维度形成的 cluster 之间构造 cluster links。

\[ \Pr(t_{il}\rightarrow t_{ik}\mid \rho_i,A_i) \propto \begin{cases} A^{i}_{lk}, & t_{ik}>t_{il},\\ \rho_i, & t_{ik}=t_{il}. \end{cases} \]

这里 \(\rho_i\) 表示 self-affinity，对应产生新 immigrant 的倾向；\(A^{i}_{lk}\) 表示历史事件触发当前事件的亲和度，通常由时间窗口项和触发核项共同决定。跨维度时，BRUNCH 再根据 cluster 间最强的触发核响应决定 cluster link。

\[ \Pr(s\rightarrow g\mid B) \propto \max_{\substack{t_{ik}\in s,\;t_{jl}\in g\\ t_{ik}\lt t_{jl}}} \left|\alpha_{ij}\right|\,\phi_{ij}(t_{jl}-t_{ik}) \]

算法流程：先根据单维事件之间的触发强度采样 event links，形成局部 cluster；再根据跨维 cluster 之间的最大触发响应采样 cluster links；最后通过 Monte Carlo 推断方法 MEDIA 在 event links 与 cluster links 之间交替采样，得到最可能的 branching structure。

对当前方法的启发：当前建模中的父事件变量 \(z_i\)、级联链路和弱因果解释，都可以看作是对 BRUNCH 分支结构思想的简化吸收：不只是预测事件是否会发生，还要解释事件之间可能的父子触发关系。

参考三：DHP——连续时间流式聚类

出处：Nan Du、Mehrdad Farajtabar、Le Song（Georgia Institute of Technology）；Amr Ahmed（Google Strategic Technologies）；Alexander J. Smola（Carnegie Mellon University）. Dirichlet-Hawkes Processes with Applications to Clustering Continuous-Time Document Streams, KDD 2015。

DHP 将 Dirichlet Process 的“自适应生成新簇”能力和 Hawkes Process 的“历史事件自激发”能力结合起来，用于连续时间事件流聚类。与需要把时间离散成 episode 的 RCRP 不同，DHP 直接在连续时间上建模，不需要预先设定固定时间片。

\[ \bar{\lambda}(t)=\lambda_0+\sum_{d=1}^{D}\lambda_{\theta_d}(t) \]

其中 \(\lambda_0\) 是产生新簇的背景强度，\(\lambda_{\theta_d}(t)\) 是已有簇 \(d\) 的 Hawkes 自激发强度。新事件既可能开启一个新簇，也可能加入某个已有簇：

\[ P(s_n=k\mid t_n,\mathrm{rest})= \frac{\lambda_{\theta_k}(t_n)}{\lambda_0+\sum_{i=1}^{n-1}\gamma_{\theta_i}(t_n,t_i)} \] \[ P(s_n=\mathrm{new}\mid t_n,\mathrm{rest})= \frac{\lambda_0}{\lambda_0+\sum_{i=1}^{n-1}\gamma_{\theta_i}(t_n,t_i)} \]

DHP 的核心思想是：簇大小不再只由历史计数决定，而是由簇级 Hawkes 强度决定。一个已有簇如果近期有较强自激发，就更容易吸收新事件；同时，\(\lambda_0\) 保证模型仍然可以产生新簇。

算法流程：对每个新到达事件，先根据内容似然与 DHP 时间先验采样簇标签；随后更新该簇的触发核参数；若同一时间戳存在多个事件，则进一步用 Gibbs / Metropolis 采样补充精确时间。整体使用 Sequential Monte Carlo 维护多个粒子，使流式聚类可以在线更新。

对当前方法的启发：DHP 说明 Hawkes 强度不仅可以解释事件触发，还可以作为连续时间聚类的动态先验：已有链路的近期活跃度越高，后续事件越容易被吸收到该链路中；同时背景强度保留新链产生的可能性。这与当前方法中“背景项开新链、触发项吸收历史链”的推理逻辑高度一致。

参考四：Latent Network Hawkes——从事件流发现隐含网络

出处：Scott W. Linderman、Ryan P. Adams（Harvard University）. Discovering Latent Network Structure in Point Process Data, ICML 2014。

该工作关注这样一个问题：很多系统的网络边并不能直接观测，只能看到节点上的事件流。论文将 Hawkes 过程与随机图模型结合，通过事件之间的相互激发来反推潜在网络结构。

\[ h_{k,k'}(\Delta t)=A_{k,k'}\,W_{k,k'}\,g_{\theta_{k,k'}}(\Delta t) \]

其中 \(A_{k,k'}\in\{0,1\}\) 表示潜在边是否存在，\(W_{k,k'}\ge 0\) 表示边强度，\(g_{\theta}\) 表示时间响应函数。这个分解把“有没有关系”和“关系有多强”分开建模。

\[ \lambda_k(t)=\lambda_{0,k}(t)+\sum_{n:s_n\lt t}A_{c_n,k}\,W_{c_n,k}\,g_{\theta_{c_n,k}}(t-s_n) \]

为了便于推断，论文利用 Poisson superposition 思想，为每个事件引入 parent variable：事件要么来自背景过程，要么来自某个历史事件产生的 impulse response。

\[ \Pr(z_n=m)\propto \lambda_m(s_n) \]

算法流程：先用随机图先验描述潜在邻接矩阵 \(A\)，用权重矩阵 \(W\) 描述相互激发强度；再为每个事件采样父事件 \(z_n\)，把观测事件分解为背景事件和触发事件；最后通过 Gibbs sampling 交替更新 \(A,W,\theta,\lambda_0,z\)，从事件流中恢复潜在网络。

对当前方法的启发：当前方法中的弱拓扑建模与它思想相近：结构关系不一定完全可靠，也不一定完全可观测；可以把结构信息作为软先验或特征，让事件数据来修正和学习最终的触发强度。

归纳来看，ISAHP 强调“实例级因果强度可以从逐项加和的强度函数中读取”，BRUNCH 强调“从点过程恢复分支结构”，DHP 强调“用连续时间 Hawkes 强度驱动流式聚类”，Latent Network Hawkes 强调“从事件流发现隐含网络”。当前方法将这四类思想统一为：用多维点过程描述事件发生，用父事件变量解释弱因果，用背景项与触发项支持在线分组，用结构特征和先验表达弱拓扑，用连续时间核函数表达弱时序。
本节给出多维时序点过程的总体框架；后续三节在此基础上展开弱拓扑、弱因果与弱时序三类「弱关联」建模。
Section 02 · Weak Topology Association

第二节：弱拓扑关联

当前算法并不把拓扑关系当作绝对可靠的硬规则，而是采用“弱拓扑”思路：拓扑信息不会直接规定某条关系一定成立，而是作为可学习的结构信号进入触发强度，并在数据稀疏时提供轻量先验；最终是否形成强触发关系，仍由事件数据和学习到的参数共同决定。

2.1 为什么是弱拓扑

在多维时序点过程里，历史事件是否会影响当前事件，取决于二者之间的触发强度。若完全不使用结构信息，模型只能依赖时间共现来解释事件间关系，容易受到偶然共现和样本稀疏的影响；若完全依赖结构信息，又会受到结构缺失、方向错误或粒度不一致的影响。

因此当前算法采用折中方式：拓扑只提供“哪些关系更可能有影响”的软信号，而不是直接规定某条边一定存在或一定不存在。

触发特征

拓扑强弱被编码成特征，进入触发强度函数，由数据学习其权重。

轻量先验

对结构相关的类型对加入 pseudo-count，在样本稀疏时提供温和支持。

2.2 触发强度：先定义要学习的关系强度

对来源类型 \(v\) 和目标类型 \(u\)，当前算法首先把二者之间的影响表示为触发强度：

\[ \alpha_{u,v}=\operatorname{softplus}\!\left(w_{\alpha}^{\top}\phi(u,v)\right) \]

这里 \(\phi(u,v)\) 是来源—目标类型对的特征，\(w_{\alpha}\) 是需要学习的参数。这个公式表达的是： 模型不直接为每一对类型记忆一个固定强度，而是通过一组可解释特征共同计算触发强度。

因此，拓扑信息和其它属性信息都不是单独决定关系是否成立，而是共同进入 \(\phi(u,v)\)，再由学习到的权重决定它们对触发强度的贡献。

2.3 拓扑特征：把结构关系转成连续信号

在定义触发强度之后，拓扑信息以特征的形式进入 \(\phi(u,v)\)。对来源类型 \(v\) 和目标类型 \(u\)，算法会计算一个有方向的拓扑分数：

\[ r_{\mathrm{topo}}(v\rightarrow u)\in[0,1] \]

它表示来源类型 \(v\) 所属实体到目标类型 \(u\) 所属实体之间的结构接近度或可达强度。例如，同一实体、直接相连、多跳可达、反向可达、无向可达等关系会被压缩成不同强度的分数。

于是，触发强度中会出现如下项：

\[ w_{\mathrm{topo}}\,r_{\mathrm{topo}}(v\rightarrow u) \]

如果学习到 \(w_{\mathrm{topo}}>0\)，说明结构更接近的类型对更容易形成触发；如果 \(w_{\mathrm{topo}}\) 较小甚至为负，说明仅靠拓扑并不能解释数据中的触发关系。

这就是“弱拓扑”的关键：拓扑以连续特征形式进入模型，但拓扑特征的权重由数据学习，而不是由人工固定。

2.4 其它静态特征：补充拓扑之外的归纳信号

除了拓扑关系，算法还利用类型关系、位置代理和实体属性来增强泛化能力。

当前其它静态特征可以概括为：

  • 类型对：不同事件类型之间的方向性触发模式。
  • 同类事件：来源与目标是否属于相同事件类型。
  • 同实体：来源与目标是否属于同一实体。
  • 同站点 / 位置代理：二者是否处于相同位置范围。
  • 同厂家：相同来源系统或实体属性带来的相似模式。
  • 同实体类型：来源与目标是否属于相同类别。
  • 交叉特征：“结构相关且位置相近”是否具有额外增强作用。

因此，最终触发强度不是由单一拓扑信息决定，而是由拓扑、类型关系、位置代理和实体属性等多种信号共同决定。

2.5 动态状态特征：上下文感知的触发

除了静态属性，真实网络中一个事件是否会引发级联告警，往往还取决于设备当前的运行状态。为此，算法在静态关联之上引入了动态状态特征（Dynamic State Features）。

在任意时刻 \(t\)，算法不仅会追踪来源实体 \(v\) 在触发发生时的未恢复告警状态，还会同时观测目标实体 \(u\) 受波及前夕的运行状态（例如：当前是否处于断电、脱网、链路中断等），将二者拼接为联合动态特征向量 \(\mathbf{m}_{u,v}(t)\)。此时，触发强度的计算升级为全量上下文感知模式：

\[ \alpha_{u,v}(t)=\operatorname{softplus}\!\left(w_{\mathrm{static}}^{\top}\phi_{\mathrm{static}}(u,v) + w_{\mathrm{dyn}}^{\top}\mathbf{m}_{u,v}(t)\right) \]

动态特征使得模型能够学习到极度复杂的条件触发逻辑。它不仅能学到“源设备处于断电状态时发出的告警破坏力更大”，还能学到“当目标设备自身已经存在未恢复的脱网或性能问题时，它更容易被相邻设备的告警所触发（即目标设备的脆弱性）”。源和目标的实时状态在这里实现了完美的双向协同。

2.6 拓扑先验：用 pseudo-count 提供弱结构偏置

在训练中，E-step 会得到每个来源—目标类型对的期望触发次数。记：

\[ N_{u,v}=\sum_i\sum_{j:u_i=u,\,u_j=v}q(z_i=j) \]

其中 \(q(z_i=j)\) 是事件 \(i\) 由历史事件 \(j\) 解释的软责任。为了让结构相关但样本较少的关系不被完全忽略，算法可以加入拓扑 pseudo-count：

\[ \widetilde{N}_{u,v}=N_{u,v}+\rho\,r_{\mathrm{topo}}(v\rightarrow u) \]

其中 \(\rho\) 是拓扑先验强度超参数。它不是要强迫拓扑关系一定成立，而是在数据不足时给予结构相关关系一个温和的初始支持；当真实事件数据足够多时，数据项 \(N_{u,v}\) 会主导学习结果。

2.7 小结

当前算法中的弱拓扑关联可以概括为两层：

\[ \mathrm{WeakTopologyAssociation} = \mathrm{TopologyFeatureScoring} + \mathrm{Static/DynamicContext} + \mathrm{TopologyPseudoCountPrior} \]

它既避免了完全忽略结构信息带来的偶然共现问题，也避免了完全相信结构信息带来的脆弱性。最终，拓扑只是提供可学习的归纳偏置，真正的触发强度仍由事件数据和学习参数共同确定。

2.8 参考文献：弱拓扑相关工作

与第二节“弱拓扑关联”最相关的工作，主要体现为两条思路：一类是把已有结构网络和潜在扩散网络分开建模，说明已有拓扑不是唯一可信的传播通道；另一类是直接利用社交网络中的 follow 关系作为传播网络骨架，再在该骨架上学习不同 aspect 下的传播强度。前者对应 Cross-layer Cascades，后者对应 MMRate。

参考一：Diffusion Network Inference for Cross-layer Cascades——观测拓扑与潜在拓扑的联合推断

出处：Siyu Huang、Yubai Yuan、Abdul Basit Adeel（The Pennsylvania State University）. Diffusion Network Inference for Cross-layer Cascades, NeurIPS 2025。

该工作关注 cross-layer cascade 场景：一个级联并不一定只沿着单一观测网络传播，而可能在观测网络和潜在网络之间交替传播。论文用 double network mixture model 同时刻画两类扩散通道：\(\Theta\) 表示受观测网络约束的扩散网络，\(\Psi\) 表示未被观测拓扑捕获的潜在扩散网络。

\[ \Theta^{c} = \bigl( Z^{c}_{1}\Theta_{\cdot 1}+(1-Z^{c}_{1})\Psi_{\cdot 1}, \ldots, Z^{c}_{N}\Theta_{\cdot N}+(1-Z^{c}_{N})\Psi_{\cdot N} \bigr) \]

其中 \(Z_i^c\in\{0,1\}\) 表示在第 \(c\) 条级联中，节点 \(i\) 的激活更像是通过观测网络 \(\Theta\) 解释，还是通过潜在网络 \(\Psi\) 解释。也就是说，同一条级联内部，不同节点可以选择不同传播层。

\[ \widehat{\pi}^{c}_{i} = P(Z_i^c=1\mid t^{(c)}) = \frac{\pi_i P_I(t_i^c;\Theta_{\cdot i})} {\pi_i P_I(t_i^c;\Theta_{\cdot i})+(1-\pi_i)P_I(t_i^c;\Psi_{\cdot i})} \]

这个后验概率可以理解为“当前节点的激活有多大概率由观测拓扑层解释”。当 \(\widehat{\pi}^{c}_{i}\) 高时，观测拓扑更能解释该传播；当它低时，模型更倾向于认为存在未观测的潜在传播关系。

该工作还对两类网络施加不同结构约束：\(\Theta\) 受观测网络 \(A\) 的 support 约束，\(\Psi\) 用低秩结构刻画潜在社会距离或隐含关系。

\[ \Theta\odot(1-A)=0, \qquad \operatorname{rank}(\Psi)\le r \]

算法流程：先根据级联时间记录构造连续时间扩散似然；再在 E-step 中估计每个节点激活属于观测网络层还是潜在网络层的后验概率；随后在 M-step 中分别更新观测网络扩散强度 \(\Theta\)、潜在网络扩散强度 \(\Psi\) 和层选择概率 \(\pi\)。为了保证可优化性，论文将低秩约束放松为 nuclear norm，并把 M-step 分解成一组凸优化问题。

与当前弱拓扑建模的关系：当前算法同样不把已有拓扑当作硬事实，而是把拓扑作为可学习的弱信号。Cross-layer Cascades 的启发在于：观测拓扑可以看作一层可靠但不完整的结构，潜在关系可以看作另一层对观测拓扑的补充；因此，弱拓扑不只是“拓扑特征打分”，还可以进一步扩展为“观测拓扑层 + 潜在拓扑层”的联合解释框架。对应到当前方法中，\(r_{\mathrm{topo}}(v\rightarrow u)\) 和拓扑 pseudo-count 类似于观测拓扑层的软偏置，而由事件数据学习出的非拓扑触发强度则承担了部分潜在关系补偿作用。

参考二：MMRate——基于 follow 关系的多方面传播网络推断

出处：Senzhang Wang、Zhoujun Li（Beihang University）；Xia Hu（Arizona State University）；Philip S. Yu（University of Illinois at Chicago）. MMRate: Inferring Multi-aspect Diffusion Networks with Multi-pattern Cascades, KDD 2014。

该工作关注社交媒体中的 multi-aspect diffusion network inference。与完全从事件数据中无约束学习传播关系不同，MMRate 在真实微博实验中直接使用社交网络中的 following relationships 构造传播网络骨架：用户是节点，follow 关系提供可能的传播边；模型再在这些边上学习不同 aspect 下的传播速率。

\[ G=\bigcup_{k=1}^{K}G_k,\qquad G_k=(N,E_k),\qquad E_k=\bigcup_{i,j}\{\alpha^{k}_{ij}\} \]

其中 \(K\) 是预设的 aspect 数量，\(\alpha^{k}_{ij}\) 表示节点 \(i\) 到节点 \(j\) 在第 \(k\) 个 aspect 下的传播速率。同一对用户可以在不同 aspect 下具有不同传播强度，因此传播网络不是单一同质图，而是多个 aspect-sensitive 子图的组合。

MMRate 同时考虑 cascade 自身的扩散模式。对 cascade \(t^c\)，节点 \(i\) 感染节点 \(j\) 的概率由两部分共同决定：一部分是 aspect-sensitive transmission rate \(\alpha^k_{ij}\)，另一部分是 cascade pattern 参数 \(\delta^c\)。

\[ f(\Delta t^c_{ij};k,\delta^c) = g_1\left(\Delta t^c_{ij};g_2(\alpha^k_{ij},\delta^c)\right) \]

算法流程：先从微博数据中抽取用户节点和 follow 关系，形成候选传播网络；再为每条 cascade 引入隐含 aspect 变量 \(z_{ik}\)，表示该 cascade 更可能属于哪个传播方面；随后用 EM 交替更新 cascade 的 aspect 分配、aspect 分布 \(\pi_k\)，以及每条 follow 边在不同 aspect 下的传播速率 \(\alpha^k_{ij}\)。

与当前弱拓扑建模的关系：MMRate 的核心启发是：已有社交拓扑可以作为传播关系的骨架，但不能直接等同于最终传播强度。follow 边只是说明“可能有传播通道”，真正的传播能力还要根据 cascade 数据学习，并且同一条边在不同 aspect 下可能具有不同强度。对应到当前方法，设备拓扑、同站点、同厂家等关系也更适合作为弱拓扑信号进入触发强度，而不是作为硬因果边直接使用。

本节为“弱拓扑关联”的独立说明，保持通用事件建模表述；后续章节可继续展开 feature 模式、动态状态特征或在线推理机制。
Section 03 · Weak Causal Association

第三节：弱因果关联

当前算法中的“因果”不是严格实验意义上的因果关系，而是基于时序触发强度得到的弱因果解释： 一个事件可以被背景项解释，也可以被历史事件解释；当直接解释不足时，在线推理还允许在固定滞后窗口内插入未观测事件， 用来补全更连贯的触发链。

3.1 在线推理：先在观测事件之间建立弱父子关系

普通在线推理阶段，算法对每个新到达事件维护历史事件集合，只在候选集合里寻找候选父事件。推理决策可以写成：

\[ \widehat{z}_i = \arg\max_{z\in\{0\}\cup\mathcal{H}_i} s_z(i) \]

其中 \(\mathcal{H}_i\) 是事件 \(i\) 的历史候选集合，\(s_z(i)\) 表示不同候选解释的得分：

\[ s_z(i) = \begin{cases} \mu_{u_i}, & z = 0 \\ \alpha_{u_i,u_j} \, g(t_i - t_j), & z = j \in \mathcal{H}_i \end{cases} \]

即 \(z=0\) 表示事件 \(i\) 依靠自身背景强度（\(\mu\)）独立发生；\(z=j\) 表示它由历史事件 \(j\) 触发，得分取决于两者的触发强度（\(\alpha\)）与时间衰减（\(g\)）。

背景解释

若 \(\mu_{u_i}\) 占优，事件 \(i\) 被视为新链起点。

触发解释

若某个 \(s_j(i)\) 占优，事件 \(i\) 被合并到父事件 \(j\) 所在链路。

3.2 为什么需要数据插补

单次前向分配有一个局限：如果真实触发链中间存在未观测事件，那么目标事件可能无法被任何已观测历史事件很好解释。 这时，模型不应只能在“强行连接到某个观测事件”和“直接开新链”之间二选一，而应该允许一种中间解释：

\[ e_a \rightarrow m \rightarrow e_i \]

其中 \(m\) 是一个未观测到的中间事件。它不是新的未知类型，而是已有事件类型中的一个潜在发生实例。 当前算法的数据插补就是假设这类 \(m\) 的存在，从而让原本不连贯的链条变得更合理。

3.3 插补目标：联合解释观测事件、缺失事件与父子关系

记观测事件集合为 \(\mathcal{O}\)，插补出的缺失事件集合为 \(\mathcal{M}\)，父事件关系为 \(Z\)。 在线插补的目标可以抽象为：

\[ p(\mathcal{M},Z\mid\mathcal{O},\Theta) \propto p(\mathcal{O},\mathcal{M},Z\mid\Theta)\,p(\mathcal{M}) \]

其中 \(\Theta\) 是已经训练好的模型参数，\(p(\mathcal{O},\mathcal{M},Z\mid\Theta)\) 衡量插补后的整条链能否被模型解释， \(p(\mathcal{M})\) 则控制不要无限制地补充缺失事件。

\[ p(\mathcal{M})\propto\exp\left(\kappa|\mathcal{M}|\right) \]

这里 \(\kappa\) 是每个缺失事件的先验惩罚。通常 \(\kappa\) 越负，模型越保守，补出的缺失事件越少；\(\kappa\) 越大，模型越容易插入中间事件。

3.4 固定滞后窗口：持续观测、动态重采样、延后提交

数据插补不是在全量序列上离线重算，而是在一个固定滞后窗口中进行。窗口内的事件结构暂时保持可修改： 父事件可以被重采样，缺失事件可以被新增或删除；当事件超过滞后时间后，其结构被提交并输出。

\[ \mathcal{W}(t)=\{e:\,t_e\ge t-L\} \]

其中 \(L\) 是固定滞后长度。窗口越长，模型越有机会发现缺失中间事件，但在线延迟也越高；窗口越短，响应更快，但补链能力更弱。

窗口内

持续重采样父事件与缺失事件，保留多种可能解释。

窗口外

事件结构被冻结，作为当前最可信的弱因果解释输出。

3.5 插补采样：新增、删除与重采样

在窗口内，算法通过局部采样不断调整解释结构。主要包括三类动作：

  • 父事件重采样：在背景项和候选历史事件之间重新选择当前事件的解释来源。
  • 缺失事件新增：当某个观测事件缺少合适父事件时，提议插入一个已知类型的缺失事件作为中间父节点。
  • 缺失事件删除：如果某个缺失事件对整体解释贡献不足，或者带来过多额外预测惩罚，则将其移除。

对一次新增缺失事件 \(m\) 的提议，可以宏观理解为比较新增前后的解释收益：

\[ \Delta \approx \underbrace{\log s(m\rightarrow i)-\log s_{\mathrm{old}}(i)}_{\text{目标事件解释收益}} + \underbrace{\log s(\operatorname{pa}(m)\rightarrow m)}_{\text{缺失事件自身可解释性}} - \underbrace{C(m)}_{\text{额外预测惩罚}} + \underbrace{\kappa}_{\text{缺失先验惩罚}} \]

这四项精确映射为：

  • 目标收益：即缺失事件触发目标事件的强度，减去目标事件原有的孤儿背景强度。
  • 自身解释：在提议阶段缺失事件暂视作由背景强度产生（随后通过 Gibbs 采样为其寻找父节点）。
  • 额外惩罚：表示新增 \(m\) 以后预期会引发但实际未观测到的事件所带来的生存积分惩罚。
  • 先验惩罚：表示模型对引入虚拟事件的固定惩罚代价（通常为负值），用以约束插补的保守性。

因此，模型不会只因为某个缺失事件能解释一个目标事件就无条件保留它；这个缺失事件自身也必须合理，并且不能带来无法兑现的额外级联预测惩罚。

3.6 输出：带不确定性的弱因果链

窗口内的采样会为每个事件积累父事件投票。当事件被提交时，模型输出当前最可信的父事件关系以及对应置信度。 因此，插补结果最好理解为“概率性弱因果解释”，而不是确定事实。

\[ \widehat{p}(\operatorname{pa}(i)=j) = \frac{\operatorname{votes}_{i,j}}{\operatorname{sweeps}_i} \]

真实观测事件和插补事件在输出中应被区分：观测事件是输入证据，插补事件是模型为了让触发链更连贯而提出的潜在中间节点。

3.7 小结

当前弱因果建模可以概括为：

\[ \mathrm{WeakCausalModeling} = \mathrm{ParentAssignment} + \mathrm{LagImputation} + \mathrm{PosteriorSummary} \]

它的核心思想是：先用学习到的触发强度解释观测事件之间的弱父子关系；当观测链不完整时，在固定滞后窗口内插入少量已知类型的缺失事件； 最后输出带有置信度的弱因果链，而不是声称恢复了唯一真实因果结构。

3.8 参考文献：缺失事件插补相关工作

第三节“弱因果关联”中的数据插补，主要借鉴了两类相关工作：一类是在传统多维 Hawkes 过程中显式对未观测事件进行后验采样，另一类是在神经点过程框架下用 particle smoothing 从未来观测中反推缺失事件。它们共同说明：缺失事件不只是噪声或异常值，而可以作为解释观测事件之间弱因果链条的潜在中间节点。

参考一：Hawkes Process Inference with Missing Data——多维 Hawkes 中的未观测事件后验采样

出处：Christian R. Shelton、Zhen Qin、Chandini Shetty（University of California, Riverside）. Hawkes Process Inference with Missing Data, AAAI 2018。

该工作关注多维 Hawkes 过程中的缺失事件推断：事件类型集合是已知的，但某些类型在某些时间区间内不可观测，甚至可以存在完全隐藏的事件类型。模型的目标是在给定部分观测事件的情况下，对未观测事件集合进行后验推断。

\[ p(x)= \exp\left(-\sum_i \Phi_{l_i,?}(T-t_i)\right) \prod_i \sum_{j\in I^0_{t_i}}\phi_{l_j,l_i}(t_i-t_j) \]

为了高效采样缺失事件，论文利用 Hawkes 过程的 branching / parent 表示，为每个事件引入父事件辅助变量；同时引入 virtual events 作为“潜在但尚未实现”的候选事件。由于采样过程中会把 virtual event 变成 real event，或者把 real event 删除回 virtual event，事件数量本身会变化，因此这里不是固定维度空间上的普通 MCMC，而是带有维度跳转思想的 Reversible Jump MCMC（RJMCMC）。

\[ p(x,a,\widetilde{x},\widetilde{a}) = \prod_i \exp\left[-(\kappa+1)\Phi_{l_i,?}(T-t_i)\right] \prod_i \phi_{l_{a_i},l_i}(t_i-t_{a_i}) \prod_i \kappa\,\phi_{l_{\widetilde{a}_i},\widetilde{l}_i}(\widetilde{t}_i-t_{\widetilde{a}_i}) \]

与传统 MCMC 相比，RJMCMC 的关键差别在于：传统 MCMC 通常在固定维度状态空间内更新变量，例如只改变某个事件的父节点；而 RJMCMC 允许在不同维度的状态空间之间跳转，例如新增一个缺失事件会增加状态维度，删除一个缺失事件会降低状态维度。论文中的 virtualness move 和 parent move 都具有这种跨维度特征：当一个 virtual event 被接受为 real event 时，模型相当于向当前解释链中新增了一个未观测事件；当某个 real event 被移回 virtual event 时，模型相当于删除了一个缺失事件。

算法流程：先固定观测事件和观测区间；再维护一个包含真实事件、缺失事件、虚拟事件和父事件指针的状态；随后通过三类 RJMCMC move 更新状态：重采样 virtual children、在 virtual / real event 之间切换、重采样 parent。第一类 move 主要刷新候选虚拟事件；第二类 move 显式改变缺失事件个数；第三类 move 在重采样父节点时也可能把某个 virtual parent 提升为 real parent。这样可以在连续时间空间中搜索可能的缺失事件，而不需要枚举无限多个候选时间点。

与当前弱因果建模的关系：当前方法中的缺失事件插补也把未观测事件看作潜在中间节点，用来解释观测事件之间无法直接连通的弱因果链。该工作提供的启发是：缺失事件应该和父事件结构一起被推断，而不是先独立补点、再事后连边；并且一旦允许“新增 / 删除缺失事件”，推理本质上就不再只是固定维度的父节点重采样，而更接近 RJMCMC 式的跨维度结构搜索。

参考二：Imputing Missing Events in Continuous-Time Event Streams——基于粒子平滑的缺失事件插补

出处：Hongyuan Mei、Jason Eisner（Johns Hopkins University）；Guanghui Qin（Peking University）. Imputing Missing Events in Continuous-Time Event Streams, ICML 2019。

该工作关注连续时间事件流中的缺失事件插补。给定完整事件流模型 \(p_{\mathrm{model}}\) 和缺失机制 \(p_{\mathrm{miss}}\)，目标是根据观测事件 \(x\) 推断缺失事件 \(z\) 的后验分布。

\[ p(z\mid x) \propto p_{\mathrm{model}}(x\uplus z)\,p_{\mathrm{miss}}(z\mid x\uplus z) \]

论文首先给出 particle filtering：按时间顺序从模型强度中采样缺失事件，并在遇到观测事件时强制将其加入序列；如果当前粒子的历史无法很好解释未来观测，就会被降低权重。为了利用未来观测信息，论文进一步提出 particle smoothing：用一个从右到左读取未来观测的 continuous-time LSTM 修正 proposal intensity。

\[ \lambda^{q}_{k}(t\mid H(t),F(t)) = f_k\left(v_k^{\top}\bigl(h(t)+B\overline{h}(t)\bigr)\right) \]

其中 \(h(t)\) 来自从左到右读取历史事件的模型状态，\(\overline{h}(t)\) 来自从右到左读取未来观测的平滑状态。这样，提议分布不仅看过去，也看未来，从而更容易在两个观测事件之间提出合理的缺失事件。

\[ w_m \propto \frac{p_{\mathrm{model}}(x\uplus z_m)\,p_{\mathrm{miss}}(z_m\mid x\uplus z_m)}{q(z_m\mid x)} \]

算法流程：先从 proposal distribution 中顺序采样多个缺失事件序列粒子；再用 importance weight 修正 proposal 与真实后验之间的偏差；最后对带权粒子集合做 consensus decoding，得到单个低风险的缺失事件预测结果。

与当前弱因果建模的关系：当前方法的 fixed-lag 插补也是在观测事件之间提出潜在中间事件，并用模型似然和缺失先验共同决定是否保留。该工作的启发是：插补不应只依赖过去事件，未来观测也能反向约束中间缺失事件；对应到在线场景，就是通过固定滞后窗口在提交前保留一段可回看范围，从而提升弱因果链补全的可靠性。

本节为“弱因果建模”的独立说明，聚焦在线推理中的数据插补；后续章节可继续展开动态状态特征、插补参数配置或可视化输出。
Section 04 · Weak Temporal Association

第四节：弱时序关联

当前算法中的“弱时序”体现在两个方面：第一，事件之间的影响不是依赖固定长度的离散时间窗口，而是通过连续时间自激发核函数随时间衰减； 第二，训练与推理阶段都允许小范围时间乱序，在时间戳存在抖动或到达顺序不完全可靠时，仍然可以建立合理的关联解释；在线推理进一步延迟提交，以等待松弛范围内可能到达的候选事件。

4.1 为什么需要弱时序

在真实事件流中，事件之间的关系通常不是严格发生在某个固定窗口内，也不是一定完全按照真实发生顺序被观测到。 如果使用硬窗口规则，例如“只要 5 分钟内发生就相关，超过 5 分钟就无关”，会带来两个问题：

  • 窗口内的偶然共现可能被误认为相关。
  • 窗口外但仍有影响的历史事件会被直接忽略。

当前算法使用连续时间点过程来表达这种弱时序关系：历史事件的影响随时间间隔平滑衰减，而不是在某个窗口边界处突然消失。

4.2 自激发不依赖固定时间窗口

对目标类型 \(u\)，时间 \(t\) 的强度由背景项和历史事件触发项共同组成：

\[ \lambda_u(t) = \mu_u + \sum_{j:t_j \lt t} \alpha_{u,u_j}\,g(t-t_j) \]

其中 \(g(\Delta t)\) 是时间衰减核函数，表示历史事件 \(j\) 对当前时刻的影响如何随时间间隔 \(\Delta t=t-t_j\) 变化。 在指数核下：

\[ g(\Delta t)=\beta\exp(-\beta\Delta t) \]

因此，历史事件的影响不是“窗口内有效、窗口外无效”，而是：

\[ s(j\rightarrow i) = \alpha_{u_i,u_j}\,\beta\exp\left(-\beta(t_i-t_j)\right) \]

时间间隔越小，触发分数越高；时间间隔越大，触发分数逐渐降低。这样可以避免固定窗口边界带来的突变。

固定窗口规则

在窗口边界前后，关系会突然从“可关联”变成“不可关联”。

连续衰减建模

历史影响随时间平滑减弱，关联强度由学习到的触发强度和衰减核共同决定。

4.3 训练与推理都允许时间乱序下的关联

标准时序触发通常要求父事件先于子事件：

\[ t_j \lt t_i \]

但在实际观测中，时间戳可能存在抖动，或者事件到达顺序与真实发生顺序不完全一致。当前算法在训练和推理两端都引入时间松弛量 \(\epsilon\)：训练阶段允许 EM 在构造候选父事件和计算责任时考虑小范围乱序；推理阶段则允许尚未提交的事件等待松弛范围内的候选父事件。

\[ t_i-t_j\ge -\epsilon \]

其中 \(\epsilon\) 是允许的最大乱序范围。若 \(t_j\) 比 \(t_i\) 稍晚，但仍在 \(\epsilon\) 范围内，模型仍允许 \(j\) 作为 \(i\) 的候选父事件。 这不是在线推理独有的规则，而是训练和推理共享的弱时序假设。

训练阶段

在候选父事件构造和 EM 责任分配中，允许小范围负时间差，并对其触发分数进行折扣。

推理阶段

在流式场景中维护 pending 区域，等待松弛范围内的候选事件，再提交最终父事件关系。

4.4 乱序关联的衰减机制

允许乱序并不意味着完全忽略时间顺序。对于负时间差：

\[ \Delta t_{ij}=t_i-t_j \lt 0 \]

算法将核函数的时间差截断到峰值附近，同时对乱序程度施加额外衰减：

\[ g_{\mathrm{slack}}(\Delta t) = g(\max(\Delta t,0))\,\ell(\Delta t) \]
\[ \ell(\Delta t) = \begin{cases} 1, & \Delta t\ge 0,\\ \exp\left(-\dfrac{\log 2}{h_{\mathrm{late}}}|\Delta t|\right), & -\epsilon\le \Delta t \lt 0. \end{cases} \]

其中 \(h_{\mathrm{late}}\) 是乱序衰减的半衰期。乱序越严重，\(\ell(\Delta t)\) 越小；超过 \(\epsilon\) 的事件不再作为候选父事件。

小范围乱序

仍可建立关联，但触发分数会被折扣。

超过松弛范围

不再作为候选父事件，避免把明显反向的事件强行连接。

4.5 在线推理中的时间松弛

在线推理中，模型不会在事件刚到达时立即固定其父事件，而是维护一个 pending 区域。只有当当前时间推进到足以覆盖松弛范围后，事件才会被提交。

\[ \operatorname{watermark}(t)=t-\epsilon \]

对于尚未越过 watermark 的事件，模型会暂缓最终决策，以等待可能稍晚到达、但时间上仍可作为父事件的候选事件。 这样可以在在线场景中兼顾实时性和乱序鲁棒性。

4.6 小结

当前弱时序建模可以概括为：

\[ \mathrm{WeakTemporalModeling} = \mathrm{ContinuousSelfExcitation} + \mathrm{TimeSlackAssociation} \]

第一部分用连续时间自激发核函数代替固定窗口规则，使历史影响随时间平滑衰减； 第二部分在训练和推理两端使用时间松弛和乱序衰减处理观测时间不准的问题，使小范围乱序事件仍有机会被关联，但不会无衰减地破坏时间方向；在线推理中的 watermark 只是这一机制在流式提交时的实现方式。

4.7 参考文献：弱时序与时间不确定性相关工作

第四节“弱时序关联”主要对应三类相关工作：第一类处理只有区间聚合计数、没有精确事件时间的 interval-censored Hawkes；第二类直接建模事件发生时间存在不确定窗口的 point process；第三类将缺失事件和带噪时间戳统一看成 data distortion，并用 likelihood-free 的 ABC 方法学习 Hawkes 参数。它们共同说明：事件时间不一定要被当作完全精确的硬输入，而可以通过区间、窗口、扰动函数或离散化核函数进行弱时序建模。

参考一：Interval-censored Hawkes Processes——只有区间计数时的 Hawkes 拟合

出处：Marian-Andrei Rizoiu、Pio Calderon（University of Technology Sydney）；Alexander Soen、Shidi Li、Lexing Xie（The Australian National University）；Leanne J. Dong（Concordia University）；Aditya Krishna Menon（Google Research）. Interval-censored Hawkes Processes, JMLR 2022。

该工作关注 interval-censored data：数据只记录每个时间区间内的事件数量，而不记录每个事件的精确发生时间。由于 Hawkes 过程的标准点过程似然需要精确事件时间，并且 Hawkes 不具备 Poisson 过程那样的独立增量性质，不能直接把区间计数当作普通 Poisson likelihood 来拟合。

\[ C(a,b]\sim \operatorname{Poisson}\bigl(\Xi(b)-\Xi(a)\bigr) \]

论文提出 Mean Behavior Poisson Process（MBPP）：用一个非齐次 Poisson 过程的确定性强度 \(\xi(t)\) 去对应 Hawkes 强度的期望行为，并建立 MBPP 参数与 Hawkes 参数之间的对应关系。其平均强度满足类似 Hawkes 自激发的卷积方程：

\[ \xi(t)=s(t)+\int_0^t \phi(t-\tau)\,\xi(\tau)\,d\tau \]

算法流程：先把原始连续事件时间退化为区间计数 \(C(o_{i-1},o_i]\)；再用 MBPP 的区间补偿量 \(\Xi(o_i)-\Xi(o_{i-1})\) 构造 interval-censored log-likelihood；随后通过解析解或数值近似估计 \(\xi(t)\) 与 \(\Xi(t)\)，最终恢复对应 Hawkes 参数。

与当前弱时序建模的关系：当前方法并不把固定窗口当作硬关联规则，而是把时间窗口更多视为计算或观测分辨率约束。Interval-censored Hawkes 的启发在于：当精确时间不可得时，仍可以在区间层面保留自激发建模，而不是退化为简单窗口计数规则。

参考二：Point Processes with Event Time Uncertainty——事件时间不确定窗口下的点过程建模

出处：Xiuyuan Cheng（Duke University）；Tingnan Gong、Yao Xie（Georgia Institute of Technology）. Point Processes with Event Time Uncertainty, arXiv 2026。

该工作直接考虑事件时间不确定：模型不再假设事件精确发生在某个时间点 \(t_i\)，而是假设只能知道事件落在一个不确定窗口内。

\[ t_i\in \Delta_i=(\Delta_i^l,\Delta_i^r] \]

在连续时间层面，模型仍然保留 Hawkes / self-exciting point process 的思想：当前事件强度由背景项和历史影响核共同决定。

\[ \lambda[k](t)=\mu+\sum_{i:t_i \lt t}k(t_i,t) \]

当不确定窗口与规则时间网格对齐时，论文进一步将连续时间影响核量化成离散矩阵；每个矩阵元素表示一个时间格内的历史事件对另一个时间格内事件概率的平均影响。

\[ K_{p,q}=\frac{1}{h^2}\int_{I_p}\int_{I_q}k(t',t)\,dt'\,dt \]

算法流程：先用不确定窗口描述事件时间观测；再在连续时间上写出带时间不确定性的点过程似然；随后把窗口对齐到时间网格，将连续影响核转化为可学习的矩阵或网络张量；最后用梯度下降或 variational inequality 进行参数估计，并给出收敛保证。

与当前弱时序建模的关系：当前方法中的 time slack 本质上也是承认“观测时间不是绝对精确”的弱时序假设。该工作的启发是：时间不确定性可以被显式建模为窗口，并进一步转化为可学习的离散影响矩阵；这比简单地随机扰动时间戳或固定窗口匹配更原则化。

参考三：ABC Learning of Hawkes Processes with Missing or Noisy Event Times——缺失或带噪时间戳下的似然自由学习

出处：Isabella Deutsch、Gordon J. Ross（University of Edinburgh）. ABC Learning of Hawkes Processes with Missing or Noisy Event Times, arXiv 2021。

该工作把缺失事件和带噪事件时间统一称为 data distortion。其核心观点是：如果直接用失真后的观测事件学习 Hawkes 参数，强度函数估计会产生明显偏差；因此需要显式建模 distortion function \(h(\cdot)\)。

\[ \lambda(t\mid H_t,\theta)=\lambda_0(t\mid\theta)+\sum_{i:t>t_i}\nu(t-t_i\mid\theta) \]

对缺失数据，\(h(t)\) 可以表示事件在时刻 \(t\) 被成功观测到的概率；对带噪时间戳，\(h(t)\) 可以表示真实发生时间到观测时间的扰动机制。由于未观测事件或真实事件时间需要被积分掉，观测似然通常不可解析：

\[ p(Y\mid\theta) \propto \int p(Y,Y_u\mid\theta) \prod_{t_i\in Y}h(t_i) \prod_{t_l\in Y_u}\bigl(1-h(t_l)\bigr)\,dY_u \]

算法流程：先从先验或 proposal 中采样 Hawkes 参数；再根据这些参数模拟完整 Hawkes 事件流；随后通过 distortion function 生成带缺失或带噪的观测事件；最后比较模拟观测与真实观测的 summary statistics，若距离足够小则接受参数。ABC-Hawkes 因而不需要直接计算不可解析似然。

与当前弱时序建模的关系：当前方法用 time slack 和 late penalty 处理小范围时间乱序，本质上是对 noisy event time 的工程化建模。ABC-Hawkes 的启发在于：如果时间噪声或缺失机制更复杂，可以显式定义 distortion function，并通过模拟—比较的方式学习参数，而不是强行相信原始时间戳完全准确。

本节为“弱时序建模”的独立说明，聚焦连续时间自激发与时间乱序鲁棒性；后续章节可继续展开参数学习、动态状态特征或推理输出。
